Hoisting

This is an example of hoisting. Hoisting is the concept that javascript "looks ahead" to find variable definitions. In the following example, it "looks ahead to the second value of the var and returns "Undefined" 


                             var name = "Baggins";

                            (function () {
                                // Outputs: "Original name was undefined"
                                console.log("Original name was " + name);

                                var name = "Underhill";

                                // Outputs: "New name is Underhill"
                                console.log("New name is " + name);
                            })();

Closures

This is an example of using closures. By nesting the function within a function it makes it protected and reusable throughout your code. 


                           function makeFunc() {
                            var name = "Brooklyn";
                            function displayName() {
                                alert(name);
                            }
                            return displayName;
                            }

                            var myFunc = makeFunc();
                            myFunc();

"use strict"

By writing 'use strict' at the start of your code javascript with throw errors for code that will either not work or work incorrectly. One example is using a leading zero in for a number. Since using a leading zero can change the number's value, 'use strict' will throw a syntax error if you try to use it. There are many different properites of 'use strict' that can be found in the resource listed below. 


                           'use strict';
                            var sum = 015 + // !!! syntax error
                                    197 +
                                    142;
